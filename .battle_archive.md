### Insights（本場洞察）

**Techniques:**
- [RED] Cross-file count verification — 用 JSON parsing 驗證文件宣稱的數字，比人工數更可靠
- [RED] 追蹤 jq operator 語意找到 array merge 的隱藏 bug（R3-01）
- [WHITE] Array-union merge pattern: `(a + b | unique)` 用在 object merge 之後，各資料型別用語意正確的 merge 策略
- [WHITE] Non-interactive detection via `[ -t 0 ]` + smart defaults based on existing state

**Patterns:**
- Documentation count claims drift from implementation（出現在 Round 1, 2, 3 每一層）
- Destructive operations hidden behind simple commands（`cp` 看起來無害但會覆蓋用戶設定）
- Multi-consumer consistency requires workflow documentation（同一個邏輯變更要改 3+ 個檔案時，只有 checklist 能防 drift）

**Reflections:**
- 「Single source of truth」不是文件宣稱，是架構設計 — settings.json + settings.local.json 的分裂是 Round 2 最大的結構性問題
- install.sh 同時服務 first-time setup 和 sync 兩個用途，是 Round 2 destructive overwrite 的根因 — 應該一開始就分離
